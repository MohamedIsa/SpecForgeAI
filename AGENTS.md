# SpecForge Workspace Project Context & Guidelines

Welcome to the **BRD Tech Stack & Ticket Extractor** codebase. This file defines the global project context, architectural guidelines, and codebase standards for this repository.

---

## 1. Project Overview & Context
This project is a monorepo containing a web and mobile system that acts as a virtual tech lead. Users can upload a Business Requirements Document (BRD) or spec. The system parses the document, validates the requirements against a target tech stack, and generates a structured ticket backlog with context-complete developer prompts.

---

## 2. Locked Monorepo Architecture
All code must be organized inside the workspace as follows:
*   `/apps/backend/`: Fastify server integrated with tRPC routers.
*   `/apps/web/`: React application using TanStack Router.
*   `/apps/mobile/`: Expo companion mobile application.
*   `/packages/config-typescript/`: Shared, strict base TypeScript configurations.
*   `/packages/design-tokens/`: Autogenerated styling constants imported from Google Stitch `DESIGN.md`.

---

## 3. Strict Development Rules
*   **Tech Stack Lock:** Fastify + tRPC, React, Expo, PostgreSQL, Terraform, DeepSeek API (AI layer), and Google Stitch UI tokens.
*   **Type Safety:** Strict mode enabled. The type `any` is strictly prohibited in TypeScript code.
*   **UI Separation:** Component styling is completely driven by Stitch design tokens. Developers must not make independent styling, layout, or cosmetic choices. All styles must map directly to tokens in `DESIGN.md`.
*   **Test Driven:** Unit or integration tests are required for all non-trivial business logic, routing, or state transitions.
